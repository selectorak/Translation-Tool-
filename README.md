# 中英翻译助手 — 项目策略与流程文档

## 1. 项目概述

一个支持**跨软件划词翻译**、**屏幕区域翻译**和**音频实时翻译**的中英双语翻译桌面工具。

- 用户在任何软件（浏览器、Word、PDF 等）中复制文字，即可自动弹出翻译结果
- 支持拖拽选择屏幕任意区域，OCR 识别 + 语义分析后翻译（无需手动输入）
- 系统音频实时捕获 → faster-whisper 语音识别 → 流式逐句翻译
- 提供完整的主窗口界面用于整体翻译

```

────────                    ────────

     │        ├── 获取源语言/目标语言设置
```

## 5. 语义分析引擎

```
┌──────────────────────────────────────────────────────┐
│              SemanticAnalyzer.analyze(text)           │
├──────────────────────────────────────────────────────┤
│                                                      │
│  输入: OCR 识别文本                                    │
│                                                      │
│  评分维度 (0-100 综合评分):                            │
│  ┌────────────────────────────────────────────────┐  │
│  │  1. 有效字符比例 (40 分)                         │  │
│  │     ≥85% → 满分 | 60-85% → 25分 | 40-60% → 10分  │  │
│  │                                                │  │
│  │  2. 词语数量 (25 分)                             │  │
│  │     ≥8 词 → 满分 | 4-7 词 → 18分 | 2-3 词 → 10分  │  │
│  │                                                │  │
│  │  3. 句子结构 (20 分)                             │  │
│  │     有标点句号/换行 → 满分                       │  │
│  │                                                │  │
│  │  4. 乱码检测 (扣分项, 最多 -20)                  │  │
│  │     ≥3 个连续非法字符 → 扣分                     │  │
│  │                                                │  │
│  │  5. UI 文字过滤 (扣分项, 最多 -15)               │  │
│  │     时间/百分比/单按钮文字 → 降权                 │  │
│  └────────────────────────────────────────────────┘  │
│                                                      │
│  返回:                                                │
│  ┌────────────────────────────────────────────────┐  │
│  │  meaningful: bool   — 是否建议翻译              │  │
│  │  score:     0-100   — 综合质量评分              │  │
│  │  confidence: str    — high/medium/low/none     │  │
│  │  lang:      str     — zh/en/mixed/unknown     │  │
│  │  reason:    str     — 判断依据说明              │  │
│  └────────────────────────────────────────────────┘  │
│                                                      │
│  阈值:                                                │
│    ≥60 → high   → 🟢 直接建议翻译                     │
│    35-60 → medium → 🟡 可翻译，但提示低置信度          │
│    15-35 → low    → 🔴 不建议，可能是乱码              │
│    <15 → none     → ⛔ 几乎确定是无效文字              │
└──────────────────────────────────────────────────────┘
```

## 9. 线程模型

```
主线程 (GUI)
    │
    ├── root.after(200, _process_queue)  ← 定时轮询任务队列
    │       │
    │       ├── "clipboard"  → _handle_clipboard_change()
    │       │                    └── 创建后台翻译线程
    │       │
    │       ├── "selection"  → _handle_selection_translate()
    │       │                    └── 创建后台翻译线程
    │       │
    │       └── "full_translate" → 同理
    │
    ├── _hide_for_screen_capture() → ShowWindow(0) + withdraw()
    │       └── RegionSelector 在主线程阻塞等待
    │              │
    │              ├── 取消 → _restore_after_screen_capture()
    │              └── 确认 → OCR 后台线程 → 确认对话框(主线程) → 翻译
    │
    └── root.mainloop()

后台线程 1: 剪贴板监控 (daemon)
    │
    └── while monitoring:
            sleep(0.6)
            text = pyperclip.paste()
            if text != last_text:
                queue.put(("clipboard", text))

后台线程 2: OCR + 翻译 (daemon, 屏幕翻译时创建)
    │
    ├── 截取屏幕区域 (mss/PIL)
    ├── OpenCV 图像预处理
    ├── Tesseract OCR 识别
    ├── 语义分析
    └── root.after(0) → 确认对话框(主线程等待) → 翻译

后台线程 3-N: 翻译任务 (daemon, 按需创建)
    │
    └── result = TranslateEngine.translate(text, from, to)
        root.after(0, lambda: update_ui(result))

后台线程 A: 音频捕获 (daemon)
    │
    └── WASAPI 环回 → 重采样 16kHz → RMS 门控 → 语音累积 → queue

后台线程 B: 语音识别 (daemon)
    │
    └── while running:
            audio = queue.get()
            result = asr.transcribe(audio, stream_callback)
            → 识别结果 queue → 翻译 queue

后台线程 C: 音频翻译 (daemon)
    │
    └── while running:
            task = translate_queue.get()
            result = engine.translate(text, from_lang, to_lang)
            → 结果 queue → UI 更新
```

## 12. 代码优化记录（2025-06）

### 12.1 模块拆分

原 `translator.py`（~70KB）拆分为多个职责单一的文件：

```
translator.py  (70KB)
    │
    ├── config.py         ← 配置常量（706B）
    ├── theme.py          ← 颜色主题（1.9KB）
    └── ui_components.py  ← FloatPopup + SettingsDialog（17.9KB）
```

| 模块 | 大小 | 拆分理由 |
|------|------|----------|
| `config.py` | 706B | 所有 magic number 集中管理，避免散落各文件 |
| `theme.py` | 1.9KB | Theme 类 + UI 切换按钮，亮色/暗色主题运行时一键切换 |
| `ui_components.py` | 17.9KB | FloatPopup（浮动弹窗）+ SettingsDialog（设置框），与主窗口逻辑解耦 |

### 12.2 翻译引擎重试机制

在 `translate_engines.py` 中新增：

- **统一 Session**：所有引擎共享 `requests.Session` 实例，连接池复用（Keep-Alive）
- **自动重试**：`HTTPAdapter + Retry`，对 408/429/500/502/503/504 自动重试 2 次
- **指数退避**：重试间隔 0.5s 基数，避免雪崩
- 引擎基类 `BaseEngine` 和 `KeyEngine` 接受 `session` 参数，逐层传递

### 12.3 日志系统

- `translator.py` 中的 `print()` → `logging.info()` / `warning()` / `error()`
- 格式：`HH:MM:SS [LEVEL] 消息`
- `audio_translator.py` 和 `screen_translator.py` 的 print 待后续迁移

### 12.4 工程化

- 新增 `requirements.txt`：锁定的依赖版本列表
- 新增 `.gitignore`：排除 `api_keys.json`、`__pycache__`、`*.wav`

### 12.5 主题切换 UI

在顶部导航栏右侧新增 **主题切换按钮**（🌙 暗色 / ☀️ 亮色）：

- **运行时递归换色**：`_toggle_theme()` → `_apply_theme_to_widget()` 递归遍历整个 widget 树，根据当前 Theme 颜色值更新 bg/fg/selectbackground 等属性
- 支持 6 类控件：`Frame`、`Label`、`Text`、`Scrollbar`、`Tk`、`Toplevel`
- 浮动弹窗（FloatPopup）与主窗口同步切换
- 状态栏提示当前主题状态

```python
# 核心逻辑
def _toggle_theme(self):
    is_dark = Theme.toggle()
    self.theme_btn.configure(text="☀️ 亮色" if is_dark else "🌙 暗色")
    self._apply_theme_to_widget(self.root)
    # 浮动弹窗也同步更新
    if self.float_popup.winfo_exists():
        self._apply_theme_to_widget(self.float_popup)
```

## 13. 设计决策记录

| 决策 | 原因 |
|------|------|
| MyMemory 作为首选 API | 国内网络环境下比 Google 更稳定 |
| 浮窗使用普通窗口而非无边框 | 支持系统级边缘拖拽缩放，用户体验更好 |
| 浮窗永不自动关闭 | 用户可能需反复查看译文，自动消失造成困扰 |
| 剪贴板结果同步到主窗口 | 便于二次翻译、朗读、复制等后续操作 |
| grid 布局 + uniform 等分 | 确保输入区和输出区始终各占一半高度 |
| queue.Queue 线程通信 | 避免 tkinter 多线程直接操作 UI 导致的崩溃 |
| daemon 线程 | 主窗口关闭时自动终止所有后台线程 |
| OpenCV auto/light/heavy 三档预处理 | 根据图像亮度/对比度自适应选择处理强度 |
| 屏幕翻译前自动隐藏窗口 | 避免主窗口和控制台遮挡待截取区域 |
| Tesseract 多路径自动检测 | 兼容默认安装、自定义路径（如 D:\tesseract） |
| 确认对话框可编辑识别文本 | OCR 难免有误，允许用户修正后再翻译 |
| 语义分析 5 维度评分 | 避免对乱码/UI控件文字进行无意义翻译 |
| 懒加载屏幕/音频翻译模块 | 不影响基础翻译功能的启动速度 |
| 音频 VAD 使用时间计数（非帧计数） | 避免说话中短暂停顿（0.3-0.5s）误触发句子切割 |
| 静音阈值 0.8 秒才切句 | 完整保留自然语速下的整句，大幅提升句子完整性 |
| 音频最大累积 15 秒 | 足够容纳长句（正常句子 5-8 秒），避免硬截断 |
| 段间重叠 1.5 秒 | 句子即使被切也能通过重叠区重新拼接 |
| beam_size=5 + best_of=5 | 大幅提升识别精度，GPU 实时比约 0.5x |
| condition_on_previous_text=True | 利用前文提升语音识别连贯性 |
| Whisper VAD silence>=350ms | 减少 Whisper 内部过度分割，配合外部 0.8s 双保险 |
| speech_pad=400ms | 句首句尾各保留缓冲，避免边界单词丢失 |
| 音频预处理静音裁剪 | 给 Whisper 干净的句子起止边界，提升准确率 |
| 上下文窗口 400 字符累积到 600 | 更长的上下文提示提升多句连续识别连贯性 |
| 默认模型 large-v3 | 最高精度，RTX 4050 6GB + int8_float16 刚好跑 |
| GPU 预检 cublas DLL | 提前发现缺失并自动回退 CPU，避免加载时报错 |
